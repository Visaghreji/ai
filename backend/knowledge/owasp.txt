OWASP Top 10 Reference:
- A01:2021-Broken Access Control: Users can access resources outside their intended permissions. Remediate using robust access control lists and principle of least privilege.
- A02:2021-Cryptographic Failures: Insecure transmission or storage of sensitive data. Remediate using strong TLS settings and secure encryption algorithms.
- A03:2021-Injection: Input data is executed as commands (SQL, OS Command, or LDAP). Remediate using parameterized queries and strict input validation.
- A04:2021-Insecure Design: Focuses on risks related to design flaws. Remediate using secure design patterns and threat modeling.
- A05:2021-Security Misconfiguration: Hardcoded secrets, default accounts, or unneeded active services. Remediate by auditing configs and hardening environments.
- A06:2021-Vulnerable and Outdated Components: Outdated libraries or frameworks. Remediate by maintaining software inventories and applying security updates.
- A07:2021-Identification and Authentication Failures: Weak session validation or weak authentication mechanisms. Remediate with multi-factor authentication (MFA).
- A08:2021-Software and Data Integrity Failures: Plugins, libraries, or modules from untrusted sources. Remediate by using signed packages and verifying checksums.
- A09:2021-Security Logging and Monitoring Failures: Insufficient logs to track breach attempts. Remediate by keeping secure, centralized event logs.
- A10:2021-Server-Side Request Forgery (SSRF): Fetching resources from user-supplied URLs without validation. Remediate using network-level allowlists.
